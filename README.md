# 事業計画 財務三表作成ツール

スタートアップの資金調達に必要な **予想損益計算書（P/L）・予想貸借対照表（B/S）・予想キャッシュフロー計算書（C/F）** を、少数のビジネス前提（売上・コスト・人件費・調達ラウンドなど）から自動で連動計算して作成できる、ローカル完結型のWebアプリです。

## 特徴

- 売上・変動費・固定費・人件費などの前提条件をスプレッドシート風グリッドで入力すると、P/L → B/S → C/Fが自動で連動計算されます。
- 直近3年間は月次、4〜5年目は年次で表示（VC向けピッチ資料の標準的な構成）。
- セルを直接編集して特定の月だけ数値を上書き可能（Excelのセル上書きと同様の挙動）。年次列を編集した場合は該当年の12ヶ月に均等分配されます。
- 各項目ごとに「発生開始月」を設定可能（例: 6ヶ月目から新サービスの売上が立ち上がる、など）。
- 調達ラウンド（実施月・調達額）を複数登録可能。
- 資金ショートの自動検知とランウェイ表示。
- 売上・営業利益・現金残高推移のグラフ表示（ブラシによる期間ズーム、軸ラベルの万・億単位の短縮表記対応）。
- Excel（.xlsx）形式でのエクスポート。
- プロジェクトデータをJSONファイルとしてバックアップ／復元可能（他端末への引き継ぎや誤操作からの復旧に利用）。
- サーバー・アカウント登録は不要。データはブラウザのlocalStorageに自動保存されます（単一プロジェクトのみ）。保存データはバージョン管理されており、アプリ更新時も自動でマイグレーションされます。

## MVPスコープ外（今後の拡張候補）

- CapEx／減価償却の明示的なモデリング
- 運転資金（売掛金・買掛金・在庫回転日数）の詳細な調整
- 複数シナリオ（楽観・悲観ケース）比較
- 複数プロジェクトの保存・切り替え
- 法人税の繰越欠損金控除

## ディレクトリ構成

```
src/
  types/        # 前提条件（入力）・計算結果（出力）の型定義
  engine/       # 計算エンジン（数式解決・P/L・B/S・C/F・ランウェイ・期間集計）
  store/        # Zustandストア・localStorage永続化
  hooks/        # 計算結果のメモ化フック
  components/
    grid/       # 前提条件入力グリッド・調達ラウンド・基本設定
    statements/ # P/L・B/S・C/F表示テーブル
    charts/     # 売上・利益・現金推移グラフ
    warnings/   # 資金ショート警告バナー
    export/     # Excel出力
    backup/     # プロジェクトデータのバックアップ／復元（JSONファイル入出力）
    layout/     # 全体レイアウト
  utils/        # フォーマット・表示行の共通ロジック
```

## 計算ロジックの前提（設計メモ）

- 内部計算は常に5年＝60ヶ月分を月次で保持し、表示時に4〜5年目を年次に集計します。
- 年次列のセルを編集すると、その年12ヶ月に均等分配して上書きされます。
- 各項目には「発生開始月（startMonth）」があり、その月より前は常に0として扱われます。発生開始月以降は、その月を数式上の0ヶ月目とみなしてシフトして計算されます（例: 成長率カーブが発生開始月から起算される）。ただしセルの直接上書き（overrides）は発生開始月の設定に関わらず常に優先されます。
- 貸借対照表はMVPスコープでは現金以外の資産・負債を持たないため、常に「資産（現金）＝資本金＋利益剰余金」でバランスします。
- 法人税はシンプル方式（税前利益×税率、赤字月は税額ゼロ、繰越控除なし）です。
- ランウェイは「直近3ヶ月の営業CF平均バーンレートに基づく残月数」、資金ショート警告は「60ヶ月の投影で実際に現金がマイナスに転じる最初の月」を検知しています。
- 前提条件データは`localStorage`にバージョン付きで永続化されます（`src/store/persistence.ts`）。データ構造を変更した際は`STORAGE_VERSION`を上げ、`migrateAssumptions`にマイグレーション処理を追加することで、既存ユーザーのデータおよびバックアップファイルの互換性を保ちます。
